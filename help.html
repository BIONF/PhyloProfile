How the input file looks like?

PhyloProfile accepts 3 kinds of input file:

(1) OrthoXML format, which are used by many popullar ortholog predictors or databases like InParanoid, Hieranoid, OMA, OrthoMCL, Panther, Roundup.

*Note: I tested the tool with an example XML file downloaded from OrthoXML website. If it does not work with your XML file, please let me know!

(2) Long format, which is a tab delimited file containing 5 columns: geneID, ncbiID (<ncbi>+taxonID. e.g. ncbi7029, ncbi3702), orthoID, var1, var2. Where var1 and var2 are variables for two additional information layers.

(3) Wide/matrix format, where rows represent genes and columns represent taxa. Each cell in the matrix contains <orthoID>#<var1>#<var2>. An unavailable value is written as NA, e.g. arath_2339_31:248814#NA#0.2, or homsa_8_41:119370#NA#NA or only NA (the same as NA#NA#NA).

*Note for matrix format:  the header of first column has to be "geneID". The header of each taxon must have this format "ncbi12345", in which 12345 is its NCBI taxon ID.

More detail? Pleas take a look at the example files test.main, test.main.long or test.main.xml in /data/demo/ :)

 

Error while parsing orthoXML input file

Currently I use a python script (data/orthoxmlParser.py) to parse orthoXML input file. This script needs 2 modules (libraries) called bs4 and lxml for working. So please install these modules if necessary (sudo pip install bs4, sudo pip install lxml).

If it still not works, you can try to run either that python script (python data/orthoxmlParser.py -i yourInput.orthoxml) or its alternative perl script (perl data/orthoxmlParser.pl -i yourInput.orthoxml) in the terminal and save the result as a new input for PhyloProfile.

 

Additional file

An additional annotation file can be provided. Since the tool initially has been made to work with protein architecture annotations, the annotation file has to have 6 columns separated by tab: (1) pairID = groupID#orthologID#seedID, (2) orthologID/seedID, (3) feature name (pfam domain, smart domain,etc.), (4) start position, (5) end position, (6) weight value (only available for seed protein)

Pleas take a look at the example files in data/demo/domains folder for more details.

 

Download function does not work

Problem: clicked on the "Download plot" (or Download filtered data) button, entered a file name on to "Download file" window and clicked Save, but the file...was not saved :(

=> Click on "Open im Browser" to open the app using internet browser. Now the download function should work.

I tested this function using Ubuntu 14.04 LTS and it worked with Firefox web browser. On a Mac machine, it always work ;)  

 

Errors while plotting

Error: arguments imply differing number of rows: 0, 1

=> does your input matrix contain only 1 line???

Error: id variables not found in data: geneID

=>  the header of gene column has to be "geneID".

Error: no rows to aggregate

=> are the first value "NA" everywhere??? If yes, please replace them by 0 or 1 or whatever but not NA! If not, you\'ve found a new bug. Please contact me ;)

Error: second argument must be a list

=> please check if any of input genes is absent in all taxa, i.e. the complete row is filled with NA. The tool, unfortunately, does not accept these genes.

Error: Error in if: missing value where TRUE/FALSE needed

=> Failed to fetch taxonomy information. This problem is caused (mostly) by missing taxonomy information from data/taxonID.list.fullRankID file. One solution could be: remove data/taxonID.list.fullRankID file and restart the tool in order to reset that taxonomy file.

 

Errors by showing Point\'s info

Error: argument is of length zero

=> re-select (super)taxon to highlight, it will work again :)

 

Errors by plotting domain architecture

Error: Aesthetics must be either length 1 or the same as the data (1): x, y, yend, xend

=> please check if the ID of selected sequence, which is shown as orthoID in Detailed plot, is present in domain input file. This error is certainly caused by the missing ID in domain input file.
